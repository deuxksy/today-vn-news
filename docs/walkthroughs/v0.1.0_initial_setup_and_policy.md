# Walkthrough - Initial Setup & AI Collaboration Policy (v0.1.0)

프로젝트 기틀을 마련하고 'Vibe Coding'을 위한 AI-Human 협업 체계를 확립한 단계입니다.

## 주요 성과 (Key Achievements)

### 1. 프로젝트 초기화
- 6대 표준 파일 구조 확정 (`README`, `ContextFile`, `GEMINI`, `CHANGELOG`, `main`, `requirements`).
- Human/AI 문서 역할 분리: 사용자는 가이드라인(`README`)을, AI는 비즈니스 로직(`ContextFile`)을 관리.

### 2. AI 협업 정책 (AI Autonomy)
- **자율 작업 권한**: `ContextFile.md`를 제외한 모든 코드 및 Git 커밋 작업을 AI가 자율적으로 수행하도록 설정.
- **Git Flow**: 수시 `commit`은 AI가, 최종 `push`는 사용자 승인을 받는 체계 구축.

### 3. 인프라 설계
- 분산 인프라 기기별 역할 정의 (NAS:저장, Deck:배치, Mac:개발).
- 데이터 흐름(Data Flow) 및 큐레이션 우선순위(건강/IT/로컬) 설정.

## 기술적 결정 (Technical Decisions)

- **Vibe Coding**: 엄격한 클래스 추상화보다 속도와 실용성 중심의 코드 작성 지향.
- **Gemini CLI 활용**: 초기 수집 모듈 구축을 위해 Gemini CLI를 우선 도입.

## 검증 (Verification)
- AI가 자율적으로 파일을 수정하고 커밋하는 워크플로우 작동 확인.
- 프로젝트 루트 구조의 간결성(Ultra-light) 유지 여부 확인.
